### Comparativa Semántica de Código: Bootstrap, Bulma y Tailwind CSS

A continuación, se presenta un cuadro comparativo que ilustra las diferencias semánticas y de sintaxis entre Bootstrap, Bulma y Tailwind CSS para la creación de componentes web de uso común.

| Componente | Bootstrap | Bulma | Tailwind CSS |
|---|---|---|---|
| **Botón** | Utiliza clases predefinidas para estilos y tamaños. Es semánticamente claro y directo.  ```html <button type="button" class="btn btn-primary">Botón Primario</button> ``` | Emplea un enfoque similar a Bootstrap, con clases descriptivas para colores, tamaños y estados.  ```html <button class="button is-primary">Botón Primario</button> ``` | Se basa en clases de utilidad para construir el estilo del botón desde cero. Ofrece gran flexibilidad a costa de un código HTML más verboso.  ```html <button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded"> Botón Primario </button> ``` |
| **Card** | Proporciona una estructura de componentes clara con clases como `.card`, `.card-body` y `.card-title`.  ```html <div class="card" style="width: 18rem;">   <img src="..." class="card-img-top" alt="...">   <div class="card-body">     <h5 class="card-title">Título</h5>     <p class="card-text">Contenido...</p>     <a href="#" class="btn btn-primary">Botón</a>   </div> </div> ``` | Ofrece una estructura similar utilizando clases como `.card`, `.card-image` y `.card-content`. Es muy legible y se basa en Flexbox.  ```html <div class="card">   <div class="card-image">     <figure class="image is-4by3">       <img src="..." alt="Imagen">     </figure>   </div>   <div class="card-content">     <div class="content">Contenido...</div>   </div> </div> ``` | No posee un componente de "card" predefinido. Se construye combinando utilidades para sombras, espaciado, bordes y tipografía.  ```html <div class="max-w-sm rounded overflow-hidden shadow-lg">   <img class="w-full" src="..." alt="...">   <div class="px-6 py-4">     <div class="font-bold text-xl mb-2">Título</div>     <p class="text-gray-700 text-base">Contenido...</p>   </div> </div> ``` |
| **Navbar** | Ofrece una barra de navegación completa y responsiva con componentes anidados y clases específicas como `.navbar`, `.navbar-brand` y `.navbar-nav`. Requiere JavaScript para su funcionalidad.  ```html <nav class="navbar navbar-expand-lg navbar-light bg-light">   <div class="container-fluid">     <a class="navbar-brand" href="#">Navbar</a>     <button class="navbar-toggler" type="button" ...>       <span class="navbar-toggler-icon"></span>     </button>     <div class="collapse navbar-collapse">       ...     </div>   </div> </nav> ``` | Proporciona una estructura modular basada en Flexbox con clases como `.navbar`, `.navbar-menu` y `.navbar-item`. No incluye JavaScript por defecto, la interactividad debe ser implementada por el desarrollador.  ```html <nav class="navbar" role="navigation">   <div class="navbar-brand">     <a class="navbar-item" href="#">Navbar</a>     <a role="button" class="navbar-burger" ...>...</a>   </div>   <div class="navbar-menu">     ...   </div> </nav> ``` | La barra de navegación se construye desde cero utilizando utilidades de Flexbox, espaciado, colores y tamaño de texto. Permite un diseño completamente personalizado.  ```html <nav class="flex items-center justify-between flex-wrap bg-gray-800 p-6">   <div class="flex items-center text-white mr-6">     <span class="font-semibold text-xl">Mi Navbar</span>   </div>   <div class="block lg:hidden">...</div>   <div class="w-full block lg:flex lg:items-center">     ...   </div> </nav> ``` |
| **Alerta** | Utiliza clases predefinidas y semánticas para mostrar mensajes, como `.alert` y `.alert-success` para indicar el propósito del mensaje.  ```html <div class="alert alert-success" role="alert">   ¡Operación exitosa! </div> ``` | Denominadas "Notifications", siguen una semántica similar a Bootstrap, con clases como `.notification` y `.is-success`.  ```html <div class="notification is-success">   ¡Operación exitosa! </div> ``` | No existe un componente de alerta. Se construye combinando utilidades para fondo, color de texto, borde, espaciado y redondeado.  ```html <div class="bg-green-100 border border-green-400 text-green-700 px-4 py-3 rounded relative" role="alert">   <span class="block sm:inline">¡Operación exitosa!</span> </div> ``` |
| **Modal** | Proporciona un componente modal completo con estructura y comportamiento definidos por clases y atributos `data-bs-*`. Depende del JavaScript de Bootstrap.  ```html <div class="modal" tabindex="-1">   <div class="modal-dialog">     <div class="modal-content">       <div class="modal-header">...</div>       <div class="modal-body">...</div>       <div class="modal-footer">...</div>     </div>   </div> </div> ``` | Ofrece la estructura HTML/CSS para el modal con la clase `.modal`. La visibilidad se controla añadiendo o quitando la clase `.is-active` mediante JavaScript propio.  ```html <div class="modal">   <div class="modal-background"></div>   <div class="modal-content">     <!-- Contenido del modal -->   </div>   <button class="modal-close is-large"></button> </div> ``` | Un modal se construye desde cero. Se requiere crear un fondo superpuesto y un contenedor para el contenido, gestionando la visibilidad con JavaScript y clases de utilidad como `hidden` o `flex`.  ```html <div class="fixed z-10 inset-0 overflow-y-auto">   <div class="flex items-center justify-center min-h-screen">     <div class="fixed inset-0 bg-gray-500 opacity-75"></div>     <div class="bg-white rounded-lg text-left ... z-20">       <!-- Contenido del modal -->     </div>   </div> </div> ``` |
| **Campo de Formulario** | Simplifica el estilo de los inputs con la clase `.form-control`. Proporciona clases adicionales como `.form-label` para una estructura organizada.  ```html <div class="mb-3">   <label for="email" class="form-label">Email</label>   <input type="email" class="form-control" id="email"> </div> ``` | Utiliza clases específicas como `.input`, `.textarea`, etc. Se estructura con contenedores como `.field` y `.control`.  ```html <div class="field">   <label class="label">Email</label>   <div class="control">     <input class="input" type="email" placeholder="e.g. alex@example.com">   </div> </div> ``` | Los inputs se estilizan individualmente con clases de utilidad para bordes, sombras de foco, espaciado y colores, lo que permite una personalización total.  ```html <div>   <label for="email" class="block text-sm font-medium text-gray-700">Email</label>   <div class="mt-1">     <input type="email" name="email" class="block w-full rounded-md border-gray-300 shadow-sm focus:border-indigo-500 focus:ring-indigo-500 sm:text-sm">   </div> </div> ``` |

### Resumen de las Diferencias Filosóficas

*   **Bootstrap**: Es un framework "orientado a componentes". Te da elementos listos para usar (`.card`, `.btn`, `.modal`) que son rápidos de implementar y tienen un aspecto consistente. Es ideal para prototipado rápido y proyectos donde no se necesita un diseño único.

*   **Bulma**: También está orientado a componentes, pero es "CSS puro". Proporciona la estructura y el estilo visual, pero deja toda la interactividad (JavaScript) en manos del desarrollador. Su sintaxis es muy legible y su sistema basado en Flexbox es moderno y potente.

*   **Tailwind CSS**: Adopta un enfoque de "utilidad primero" (utility-first). No te da componentes, sino las herramientas (clases de bajo nivel) para construir cualquier componente que imagines. Esto ofrece una flexibilidad y personalización sin igual, pero a cambio de clases más largas en el HTML y una curva de aprendizaje inicial para memorizar las utilidades.
