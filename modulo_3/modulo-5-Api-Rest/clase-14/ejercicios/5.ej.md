### Actividad 5: Identificación de Errores Comunes

**Consigna:**
Analiza las siguientes URIs y métodos HTTP, identifica qué errores rompen las convenciones REST y explica cómo corregirlos:

1.  `GET /obtenerUsuario/25`
2.  `POST /actualizarProducto/10`
3.  `DELETE /usuarios?userId=5`
4.  `GET /libros/ciencia-ficcion` (para filtrar por género).

1. GET /obtenerUsuario/25
**Error:** Uso de verbo "obtener" en la URI.
**Corrección:** GET /user/25

2. POST /actualizarProducto/10
**Error:** Uso de verbo "actualizar" en la URI y uso incorrecto de POST para una actualización.
**Corrección:** PATCH /product/10 (o PUT).

3. DELETE /usuarios?userId=5
**Error:** Uso de Query Params (?userId=5) para identificar un recurso único para eliminar.
**Corrección:** El ID debe ser parte de la ruta/path: DELETE /user/5 

4. GET /libros/ciencia-ficcion
**Error:** Ambigüedad. Parece que "ciencia-ficcion" fuera el ID de un libro específico.
**Corrección:** Usar Query Params para filtrar características: GET /books?genre=ciencia-ficcion
