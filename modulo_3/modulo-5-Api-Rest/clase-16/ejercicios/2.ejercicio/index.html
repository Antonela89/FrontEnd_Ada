Conclusiones

  • Colección: Devuelve un Array [ ... ]. Usamos limit y offset para no traer miles de registros.
  • Recurso: Devuelve un Objeto { ... }. La URL termina en el ID específico.
  • Estructura: La colección devuelve un Array [] que contiene múltiples objetos. El recurso único devuelve directamente un Objeto {}.
  • Tamaño: La colección es mucho más pesada porque trae todos los registros (o una página de ellos). El recurso único es ligero.
  • Uso: La colección se usa para listados (grids, tablas). El recurso único para páginas de "Detalle de producto".
